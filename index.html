Technical Documentation

Автоматизація рекурсивного копіювання Google Drive
Платформа n8n
Архітектура Micro-workflows

Мета

Створення автоматизованого процесу для рекурсивного копіювання вмісту папок Google Drive зі збереженням ієрархії, перейменуванням файлів, динамічною обробкою зображень (додавання водяного знака) та генерацією фінального звіту в Telegram та Google Диск.

1. Як запустити воркфлоу

  • Імпорт: Імпортуйте обидва JSON-файли (Main та Sub) у ваш інстанс n8n.
  • Активація підпроцесу: Відкрийте Sub-Workflow, збережіть його та обов'язково переведіть тумблер у верхньому правому куті в стан Active. Скопіюйте його Workflow ID з URL-адреси.
  • Зв'язування: Відкрийте Main Workflow, у вузлі Execute Sub-workflow вставте скопійований ID у поле Workflow ID.
  • Налаштування конфігурації: У вузлі Config вкажіть стартові параметри.
  • Запуск: Натисніть кнопку Execute Workflow.

2. Необхідні Credentials

Google Drive OAuth2 API

  • Читання, створення файлів/папок
  • Пошук ієрархії
  • Завантаження зображень

Telegram API

  • Bot Token (BotFather)
  • Відправка фінального звіту

3. Логіка роботи

Main Workflow (Основний процес)

  1. Ініціалізація: Зчитування налаштувань та створення нової папки у Google Drive.
  2. Створення Data Table: Тимчасова таблиця в базі n8n для безпечного збору статистики без навантаження на API.
  3. Виклик: Запуск Sub-Workflow рекурсії.
  4. Агрегація: Вивантаження всіх зібраних типів файлів та агрегація Code-вузлом.
  5. Звітність: Створення .txt файлу та відправка лога в Telegram.

Sub-Workflow (Рекурсія та обробка)

  • Вузол Loop для послідовної обробки файлів (захист від Out of Memory).
  • Маршрутизація (Switch):
    • Папки: Викликає сам себе (рекурсія) → запис у Data Table.
    • Зображення: Завантажується → Edit Image (динамічні формули) → запис у Data Table.
    • Інше: Пряме копіювання.

4. Обмеження та масштабування

Ліміти API (Rate Limiting): У поточній конфігурації процес виконується на максимальній швидкості. При масивних копіюваннях можлива помилка 429 Too Many Requests.

Рекомендація для Production: Додати вузол Wait (1 сек) в кінець циклу Loop у підпроцесі, а також увімкнути Retry On Fail (3 спроби) у налаштуваннях вузлів Google Drive для забезпечення відмовостійкості.
