Це дуже цінується на співбесідах.Документація: Автоматизація рекурсивного копіювання Google DriveПлатформа: n8nМета: Створення автоматизованого процесу для рекурсивного копіювання вмісту папок Google Drive зі збереженням ієрархії, перейменуванням файлів, динамічною обробкою зображень (додавання водяного знака) та генерацією фінального звіту в Telegram та Google Диск.1. Як запустити воркфлоуПроцес складається з двох пов'язаних воркфлоу: Main Workflow (основний контролер) та Sub-Workflow (рекурсивний обробник).Імпорт: Імпортуйте обидва JSON-файли (Main та Sub) у ваш інстанс n8n.Активація підпроцесу: Відкрийте Sub-Workflow, збережіть його та обов'язково переведіть тумблер у верхньому правому куті в стан Active. Скопіюйте його Workflow ID з URL-адреси.Зв'язування: Відкрийте Main Workflow, у вузлі Execute Sub-workflow вставте скопійований ID у поле Workflow ID.Налаштування конфігурації: У вузлі Config (початок Main Workflow) вкажіть стартові параметри:URL вихідної папки.Бажану назву для нової папки.Запуск: Натисніть кнопку Execute Workflow у Main Workflow.2. Необхідні креденшіали (Credentials)Для успішної роботи автоматизації необхідно налаштувати два підключення:Google Drive OAuth2 API:Необхідні права: Читання, створення та модифікація файлів/папок.Використовується для пошуку, копіювання, завантаження оброблених зображень та створення файлу звіту.Telegram API:Необхідний Bot Token, отриманий через BotFather.Використовується для відправки фінального текстового звіту в месенджер.3. Опис кроків та логіка роботиАрхітектура побудована за принципом мікросервісів з використанням локальної бази даних n8n для агрегації статистики.Main Workflow (Основний процес)Ініціалізація: Зчитування налаштувань (Config) та створення нової цільової папки (Root Destination Folder) у Google Drive.Створення Data Table: Створення тимчасової таблиці в базі даних n8n (copy_stats) для безпечного збору статистики без навантаження на API Google.Виклик рекурсії: Запуск Sub-Workflow з передачею ID вихідної папки, ID нової папки та назви тимчасової таблиці.Агрегація даних: Після завершення рекурсії вузол Data table (Get Rows) вивантажує всі зібрані типи файлів. Вузол Code підраховує кількість файлів, папок та групує їх за MIME-типами.Звітність: Створення текстового файлу copy_info_[name]_[date].txt у новій кореневій папці та відправка підсумкового повідомлення в Telegram.Очищення: Видалення тимчасової Data Table для звільнення пам'яті.Sub-Workflow (Рекурсія та обробка)Пошук: Отримання вмісту поточної директорії ('ID' in parents).Троттлінг (Loop): Використання вузла Loop для послідовної обробки файлів (по 1 шт.), що запобігає помилці Out of Memory (OOM) при завантаженні великих файлів.Маршрутизація (Switch/If):Папки: Створюється нова підпапка $\rightarrow$ вузол Execute Workflow викликає сам себе (рекурсія) для занурення всередину $\rightarrow$ запис у Data Table.Зображення: Файл завантажується $\rightarrow$ вузол Get Information збирає метадані (ширина/висота) $\rightarrow$ вузол Edit Image накладає текст $\rightarrow$ файл завантажується в Диск $\rightarrow$ запис у Data Table.Інші файли: Пряме копіювання через Google Drive API $\rightarrow$ запис у Data Table.Технічне рішення (Edit Image): Оскільки стандартний вузол n8n не підтримує нативне вирівнювання "по центру", координати X та Y, а також розмір шрифту (5% від ширини) розраховуються динамічно через математичні Expressions залежно від роздільної здатності кожного окремого фото.4. Обмеження та рекомендації до масштабуванняЛіміти Google Drive API (Rate Limiting): У поточній конфігурації процес виконується на максимальній швидкості сервера. При копіюванні директорій з сотнями файлів Google може повернути помилку 429 Too Many Requests.Рекомендація для Production: Додати вузол Wait (1 секунда) в кінець циклу Loop у підпроцесі, а також увімкнути опцію Retry On Fail (3 спроби) у налаштуваннях вузлів Google Drive для забезпечення відмовостійкості.Глибина рекурсії: Максимальна глибина вкладеності папок обмежена системними налаштуваннями n8n для вкладених підпроцесів (зазвичай безпечна глибина — до 15-20 рівнів).Обробка великих зображень: Вузол Edit Image обробляє файли в оперативній пам'яті. Фотографії розміром понад 50-70 МБ можуть викликати короткочасні стрибки споживання RAM.
